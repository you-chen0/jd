# 禁毒连连看 — 移动端适配方案

## 一、当前问题诊断

测试环境：375px 宽 iPhone（最坏情况）

### 1.1 卡片溢出（严重）

`buildGrid` 中卡片尺寸硬编码为固定 px：

```js
var sz = {4:'100px',5:'76px',6:'64px'}[cols];
```

第三关 18 对 = 36 张卡片，6 列布局：

| 项目 | 计算 | 结果 |
|------|------|------|
| 卡片总宽 | 6 × 64px | 384px |
| 间隙总宽 | 5 × 6px | 30px |
| 网格总宽 | | **414px** |
| 实际可用 | 375 − 36(padding) | **339px** |
| 溢出量 | | **75px** |

第一关（5 列 76px 卡片）：5×76 + 4×9 = 416px，同样溢出。

### 1.2 可用宽度不足

```css
.app-shell {
  max-width: 680px;
  padding: 24px 18px 40px;  /* 左右共 36px */
}
```

375px 屏幕实际内容区仅 339px，已有间距偏大。

### 1.3 触摸反馈缺失

当前所有卡片交互仅依赖 `:hover` 伪类。移动端 `:hover` 行为不可靠——iOS 需要二次点击才能触发，体验割裂。缺少 `:active` 状态的即时按压反馈。

### 1.4 聚光灯不兼容触屏

```js
el.addEventListener('mouseenter', ...);
el.addEventListener('mouseleave', ...);
```

触摸设备无 mouse 事件，`#spotlight` 元素在移动端完全闲置，浪费性能。

### 1.5 刘海屏未适配

无 `env(safe-area-inset-*)` 处理。iPhone X 及后续机型底部横条会遮挡按钮和关闭链接。

### 1.6 横屏未优化

当前全竖屏假设。横屏时：
- 第三关 36 张卡片 6 列溢出
- 浏览器 chrome 占据更多纵向空间
- 卡片网格可用高度严重不足

### 1.7 第三关难题

18 对 = 36 张卡片，6 列 × 6 行。在 375px 屏上卡片势必缩至极小（~53px），内部文字不可读。这是设计上的根本矛盾——卡片数量 vs 屏幕尺寸。

## 二、修复方案

### 2.1 P0：动态卡片尺寸（核心修复）

**思路**：放弃硬编码 px 值，改为根据 `window.innerWidth` 实时计算。

```js
// buildGrid 中替换硬编码 sz
function calcCardSize(cols, gap) {
  const padding = 36;  // app-shell 左右 padding
  const maxWidth = 680;
  const vw = Math.min(window.innerWidth - padding, maxWidth - padding);
  const avail = vw - (cols - 1) * gap;
  return Math.floor(avail / cols);
}

var cols = total <= 8 ? 4 : total <= 20 ? 5 : 6;
var gap = cols === 4 ? 10 : cols === 5 ? 7 : 5;
var sz = Math.min(calcCardSize(cols, gap), 100) + 'px';  // 上限 100px
```

**效果**：无论屏幕宽度如何，卡片总能填满可用空间而不溢出。

### 2.2 P1：触摸交互补全

**CSS 侧**：为所有可交互元素添加 `:active` 状态。

```css
.evidence-card:active {
  transform: scale(0.94);
  transition: transform 0.08s ease;
}

.btn:active {
  transform: scale(0.94);
}

.btn-quiz-opt:active {
  background: rgba(184,160,110,0.12);
}
```

**JS 侧**：检测触屏设备，跳过 spotlight 绑定。

```js
const isTouchDevice = 'ontouchstart' in window || navigator.maxTouchPoints > 0;
if (!isTouchDevice) {
  el.addEventListener('mouseenter', ...);
  el.addEventListener('mouseleave', ...);
}
```

**CSS 侧**：触屏设备隐藏 spotlight。

```css
@media (hover: none) {
  .spotlight { display: none; }
}
```

### 2.3 P1：安全区域适配

```css
.app-shell {
  padding-left: max(18px, env(safe-area-inset-left));
  padding-right: max(18px, env(safe-area-inset-right));
  padding-bottom: max(40px, env(safe-area-inset-bottom));
}

/* 底部固定元素避开横条 */
.modal-panel {
  margin-bottom: env(safe-area-inset-bottom);
}
```

同时设置 viewport 的 `viewport-fit=cover` 以启用安全区域 API：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover">
```

### 2.4 P2：第三关降列策略

当屏幕宽度不足以容纳 6 列时，第三关自动降为 4 列：

```js
// buildGrid 中
if (total > 24 && window.innerWidth < 420) {
  cols = 4;  // 36 张 → 4 列 × 9 行
}
```

4 列方案在 375px 屏上：每卡约 78px，文本清晰可读，9 行可在游戏区域内纵向滚动。虽然需要滚动，但比 6 列 53px 不可读的卡片要好。

### 2.5 P2：横屏适配

```css
@media (max-width: 750px) and (orientation: landscape) {
  .app-shell {
    padding: 8px 12px 24px;
    max-width: 100%;
  }
  .main-title {
    font-size: 1.4em;
    margin-bottom: 2px;
  }
  .subtitle {
    margin-bottom: 8px;
    font-size: 0.6em;
  }
  .progress-track {
    margin-bottom: 8px;
  }
  .stats-bar {
    margin-bottom: 8px;
  }
  .stat-cell {
    padding: 6px 4px;
  }
  .stat-val {
    font-size: 1.1em;
  }
  .grid {
    max-height: 62vh;
    overflow-y: auto;
  }
  /* 第三关强制 6 列充分利用横屏宽度 */
  .case-select { flex-direction: row; }
}
```

### 2.6 P3：触控区域增强

所有按钮和可点击元素确保最小触控区域 ≥ 44px（Apple HIG 标准）：

```css
.btn {
  min-height: 44px;
  min-width: 44px;
}

.btn-quiz-opt {
  min-height: 48px;
  padding: 14px 16px;
}

.back-link {
  min-height: 44px;
  display: inline-flex;
  align-items: center;
}
```

### 2.7 P3：响应式字体微调

在 ≤460px 断点下微调：

```css
@media (max-width: 460px) {
  .briefing-text { font-size: 0.82em; line-height: 1.7; }
  .quiz-title { font-size: 0.95em; }
  .quiz-question { font-size: 0.82em; }
  .card-label { line-height: 1.2; }
  .result-score-num { font-size: 2.4em; }
  .feedback-pop { font-size: 1.6em; }
}
```

## 三、改动清单

| 文件 | 位置 | 改动内容 | 工作量 |
|------|------|----------|--------|
| HTML | `<meta>` | 添加 `viewport-fit=cover` | 1 行 |
| CSS | 根部 | 新增 `@media (hover: none)` 块 | 5 行 |
| CSS | `.evidence-card` | 添加 `:active` 状态 | 4 行 |
| CSS | `.btn` / `.btn-quiz-opt` | 添加 `:active` 状态 + `min-height` | 6 行 |
| CSS | `.app-shell` | 添加 `safe-area-inset` | 4 行 |
| CSS | `@media (max-width: 460px)` | 补充字体、卡片、横屏规则 | 20 行 |
| CSS | 新增 | `@media (orientation: landscape)` 横屏规则 | 25 行 |
| JS | `buildGrid` | 动态计算卡片尺寸 + 降列逻辑 | 15 行 |
| JS | `buildGrid` | 触屏检测跳过 spotlight 绑定 | 5 行 |

**总计**：约 85 行改动，主要集中在 CSS 响应式断点和 JS 的 `buildGrid` 函数。

---

## 四、不做的

- **不做 PWA / Service Worker**：禁毒教育场景通常不需要离线缓存
- **不做横屏强制锁定**：横屏虽不好用但不应禁止，提供降级适配即可
- **不做图片懒加载 / srcset**：本地文件无带宽优化需求
- **不做触觉反馈（Haptic）**：`navigator.vibrate` 兼容性差且体验增益有限
